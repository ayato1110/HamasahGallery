# AGENTS.md — Hamasah Gallery

This is a Vite + React SPA exported from a Figma Make file (`@figma/my-make-file`). It is a gallery/ordering site for a custom bouquet business in Jambi, Indonesia.

## Commands

- `npm i` — install from `package-lock.json`
- `npm run dev` — Vite dev server
- `npm run build` — production build to `dist/`
- No lint, typecheck, or test commands exist

## Project structure

```
src/
  main.tsx                  — entrypoint (renders <App /> into #root)
  app/
    App.tsx                 — <RouterProvider>
    routes.tsx              — react-router v7 layout + all page routes
    pages/                  — one file per route: Home, Produk, Galeri, CaraPesan, TentangKami, Kontak, PertanyaanUmum, Blog, BlogDetail
    components/
      ui/                   — shadcn/ui components (generated by Figma Make)
      figma/ImageWithFallback.tsx  — <img> with error fallback
      Navbar.tsx, Footer.tsx, GoogleMapEmbed.tsx
    data/
      blogPosts.tsx         — blog data + getBlogPost() / getRelatedPosts() helpers
  assets/
    images.ts               — barrel file importing every .webp; import images through this, never by raw path
  assets bckup/             — stale backup of assets; do not use
  imports/                  — generated Figma Make output; do not edit manually
  styles/
    fonts.css               — Google Fonts: Style Script, Playfair Display, Inter
    tailwind.css            — Tailwind v4 entrypoint (uses @source, no PostCSS config needed)
    theme.css               — CSS custom properties + @theme inline + @layer base typography
```

## Architecture & framework quirks

- **react-router v7** — imports from `'react-router'`, not `'react-router-dom'`
- **Tailwind CSS v4** — via `@tailwindcss/vite` plugin. No PostCSS config file is needed. `tailwind.css` uses `@source` to scan `src/` and `@import 'tw-animate-css'` for animations.
- **Path alias** — `@` maps to `src/` (configured in `vite.config.ts`)
- **Figma asset resolver** — imports prefixed with `figma:asset/` are resolved to `src/assets/` by a custom Vite plugin; do not remove it
- **`cn()` utility** — in `src/app/components/ui/utils.ts` uses `clsx` + `tailwind-merge`; use it for conditional Tailwind classes
- **No CSS modules** — all styling is inline Tailwind utility classes; no `*.module.css` files exist
- **`next-themes`** is installed but unused in source (likely a shadcn/ui dependency)

## Style conventions

- Brand font: `Style Script` (cursive), used for "Hamasah Gallery" heading
- Accent color: `#D97706` (amber), for links, underlines, and decorative elements
- Two-space indent, single quotes, semicolons
- Components: PascalCase files, named exports
- Use `ImageWithFallback` for all `<img>` elements (handles loading errors gracefully)

## Don't touch

- `src/imports/` — auto-generated Figma Make output
- `src/assets bckup/` — stale backup
- `guidelines/` — default Figma Make template, unconfigured
- `postcss.config.mjs` — intentionally empty (Tailwind v4 Vite plugin handles it)
